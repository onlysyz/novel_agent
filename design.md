# NovelForge — Architecture Design

## Overview

NovelForge is a desktop application that autonomously generates a complete novel from a seed concept. It uses a dual-agent system: a **Writer Agent** drafts chapters and a **Review Agent** evaluates and requests revisions.

## Core Inspiration

Based on [NousResearch/autonovel](https://github.com/NousResearch/autonovel), which successfully wrote a 79,456-word novel "The Second Son of the House of Bells" in 19 chapters.

---

## 1. Five-Layer Novel Architecture

The system maintains consistency through layered propagation:

| Layer | File | What it defines |
|-------|------|----------------|
| 5 | `voice.md` | **HOW** to write — prose style, rhythm, vocabulary limits |
| 4 | `world.md` | **WHAT** exists — geography, history, magic systems, politics |
| 3 | `characters.md` | **WHO** acts — profiles, relationships, arcs, speech patterns |
| 2 | `outline.md` | **WHAT HAPPENS** — chapter beats, plot structure, foreshadowing ledger |
| 1 | `chapters/ch_NN.md` | **THE ACTUAL PROSE** — scene-by-scene writing |

**Bidirectional flow**:
- **Top-down**: Lore → Outline → Chapters (AI generates)
- **Bottom-up**: Writing reveals gaps → Update lore/outline (AI revises)

### Layer 5: Voice (`voice.md`)

Defines the prose style. Generated via `voice_fingerprint.py` by analyzing sample texts provided by the author. Contains:
- Sentence rhythm preferences
- Vocabulary wells (words to use heavily)
- Forbidden words/phrases
- POV constraints
- Dialogue rhythm rules

### Layer 4: World (`world.md`)

The world bible. Generated by `gen_world.py`. Contains:
- Geography and climate
- Political structures
- Economic systems
- Religion and culture
- **Hard magic rules** (explicit costs, limits)
- History (only relevant to present)
- Interconnected lore (each element references others)

### Layer 3: Characters (`characters.md`)

Character registry. Generated by `gen_characters.py`. Each entry includes:
- Physical description
- Psychological profile (core wound, lie they believe)
- Speech patterns and vocabulary
- Goals (surface +深层)
- Relationships with other characters
- Arc trajectory
- Role in plot

### Layer 2: Outline (`outline.md`)

The story blueprint. Generated by `gen_outline.py` using Save the Cat beats + Sanderson plotting. Contains:
- 22-26 chapters with word count targets (~3000-4000 words each)
- Act structure (Act I: 0-23%, Act IIa: 23-50%, Act IIb: 50-77%, Act III: 77-100%)
- Per-chapter: POV, location, beat designation, emotional arc, try-fail cycle
- **Foreshadowing ledger**: 15+ threads with plant → reinforce → payoff tracking
- Minimum 3-chapter distance between plant and payoff

### Layer 1: Chapters (`chapters/`)

Sequential prose files. Written by `draft_chapter.py`. Each chapter:
- ~3000-4000 words
- Includes scene beats from outline
- Maintains continuity with previous chapter
- Seeds foreshadowing elements
- Integrates world/lore naturally

## 2. Foundation Phase

Generates all planning documents before writing begins.

```
seed.txt → [gen_world] → world.md (loop until score > 7.0)
                   ↓
          [gen_characters] → characters.md
                   ↓
          [gen_outline] → outline.md (loop until score > 7.5)
                   ↓
          [gen_canon] → canon.md (hard facts cross-referenced)
                   ↓
          [voice_fingerprint] → voice.md (author style analysis)
```

**Scoring**: Each generator has a minimum score threshold. Iterates with feedback until threshold is met (typically 5-15 iterations).

### Canonical Facts (`canon.md`)

Generated by `gen_canon.py`. A hard-fact registry that all subsequent writing must obey. Includes:
- Character birth dates
- Political alliances (exact dates)
- Geographic facts
- Timeline of major events
- Rules of magic (verified consistent)
- Character knowledge boundaries

## 3. Drafting Phase

Sequential chapter writing with retry logic.

### Pipeline: `draft_chapter.py`

**Input Assembly**:
```
1. voice.md (full text)
2. world.md (relevant sections)
3. characters.md (relevant profiles)
4. outline.md (current chapter beats + next chapter opener)
5. canon.md (relevant facts)
6. Previous chapter's last 2000 characters (for continuity)
```

**Writing Prompt** includes:
- 24 explicit rules (POV, pacing, dialogue, sensory, patterns to avoid)
- Triadic list avoidance
- Passive negative avoidance
- Over-explanation rules
- Predictable ending rules

**Output**: ~3,200 words saved to `chapters/ch_NN.md`

### Evaluation: `evaluate.py`

9-dimension scoring system:

| Dimension | What it checks |
|-----------|---------------|
| `voice_adherence` | Prose matches voice.md, sentence rhythm, vocabulary |
| `beat_coverage` | All outline beats hit and *dramatized* |
| `character_voice` | Dialogue tags removable, characters sound distinct |
| `plants_seeded` | Foreshadowing placed naturally |
| `prose_quality` | Sentence variety, specificity, show-don't-tell |
| `continuity` | Logical and emotional flow from previous chapter |
| `canon_compliance` | Facts match canon.md (1 major violation caps score at 6) |
| `lore_integration` | World does work, not set dressing |
| `engagement` | Page-turn tension, unexpected moments |

**Scoring Calibration**:
- 9-10: Publishable, competes with named fantasy works
- 7-8: Strong, needs editorial polish
- 5-6: Functional but flat, needs revision
- 3-4: Significant problems
- 1-2: Not usable

**Mechanical Slop Detection** (`slop_score()`):
- Tier 1 banned words (delve, utilize, paradigm) — up to 4pt penalty
- Tier 2 suspicious word clusters — up to 2pt
- Tier 3 filler phrases — up to 2pt
- AI tells (eyes widened, heart pounded) — up to 2pt
- Show-don't-tell violations — up to 1.5pt
- Em dash density (>15/1000 words) — up to 1pt
- Low sentence variation (CV < 0.3) — 1pt

**Retry Logic**: Chapter is redrafted up to 5 times if score < 6.0.

## 4. Review System

### Layer 1: Mechanical Checks (always on)

Regex-based anti-pattern scanner from `ANTI-PATTERNS.md`:
- Triadic listing detection
- Passive negative overuse
- Cataloging interiority
- Simile density
- Section break overuse
- Uniform paragraph length
- Predictable emotional arcs
- Rhetorical dialogue formulas
- Scene-summary ratio enforcement

### Layer 2: Reader Panel (`reader_panel.py`)

4 simulated reader personas evaluate each chapter:
1. **The Genre Fan** — Checks genre conventions
2. **The Literary Reader** — Evaluates prose craft
3. **The Continuity Hunter** — Flags plot holes
4. **The Emotional Reader** — Validates character arcs

### Layer 3: Opus Deep Review (`review.py`)

Claude Opus reviews as two personas:

1. **Literary Critic** (newspaper style) — Evaluates prose quality
2. **Professor of Fiction** — Provides specific, actionable suggestions

**Review Parsing** extracts:
- Star rating (1-5)
- Severity: major / moderate / minor
- Fix types: compression, addition, mechanical, structural, revision
- Qualified items (hedged suggestions)

**Stopping Conditions**:
- Rating ≥ 4.5 with zero major items
- Rating ≥ 4 with >50% qualified items
- ≤ 2 total items identified

## 5. Revision Phase

Multi-cycle revision process (3-6 cycles):

```
Adversarial Edit (cut 2000-3000 words via adversarial_edit.py)
         ↓
Reader Panel Evaluation
         ↓
Parse Consensus Issues
         ↓
Structural Fixes (consensus items)
         ↓
Targeted Improvements (evaluation callouts)
         ↓
Polish Pass
         ↓
Opus Review Loop ←── (revisits until stopping conditions met)
```

### Adversarial Editing (`adversarial_edit.py`)

Aggressively cuts content to force creativity:
- "Find and cut 500 words that are doing nothing"
- Identifies padding, redundant explanations, over-wound prose

## 6. State Management

### `state.json`

Persistent pipeline state:

```json
{
  "phase": "drafting",
  "current_chapter": 5,
  "chapter_scores": {
    "ch_01": 7.2,
    "ch_02": 6.8,
    ...
  },
  "revision_cycles": 2,
  "last_updated": "2024-01-15T10:30:00Z"
}
```

### Version Control

- Each chapter retains revision history
- Author can rollback to any previous version
- Git integration (commits after each chapter)

```
Manuscript → LaTeX → PDF (print-ready)
           → ePub (ereader)
           → TXT (plain text)
           → Cover Art (fal.ai)
           → Audiobook (ElevenLabs, multi-voice)
```

### Cover Generation (`gen_cover_print.py`)

- Vector ornaments
- Author-provided concept or AI-generated
- fal.ai integration for image generation

### Audiobook (`gen_audiobook.py`)

- Multi-voice (different voices per character)
- Chapter-by-chapter processing
- ElevenLabs API integration

## 8. Anti-Patterns (Writing Rules)

From `ANTI-PATTERNS.md`:

**Core Principle**: Show, don't tell. Trust the image, the gesture, the silence.

| Rule | Description |
|------|-------------|
| The Over-Explain | Cut paragraphs analyzing gestures/emotions |
| Triadic Listing | Avoid "X. Y. Z." — combine two items |
| Negative-Assertion | Max 1 "He did not..." per chapter |
| Cataloging-by-Thinking | Real interiority is messy; use fragments |
| The Simile Crutch | Max 2 "the way X did Y" per chapter |
| Section Break Overuse | Max 2 breaks per chapter |
| Uniform Paragraphs | Vary: 1-2 sentences for impact, 6+ for building |
| Predictable Arcs | Include 1 surprising moment per chapter |
| Repetitive Endings | No two chapters end the same way |
| Rhetorical Dialogue | Vary structures; characters shouldn't speak in epigrams |
| Over-Polished Dialogue | Include false starts, interruptions |
| Scene-Summary Ratio | 70%+ in-scene; summary only for time compression |
